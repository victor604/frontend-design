# Contributing to Frontend Design

Thank you for your interest in contributing! This skill helps AI assistants create beautiful, memorable web interfaces.

## Ways to Contribute

### 1. Report Issues
- Found a bug? [Open an issue](https://github.com/YOUR_USERNAME/frontend-design/issues)
- Have a suggestion? Share your ideas!
- Unclear documentation? Let us know!

### 2. Add Aesthetic Directions
We love new design styles! To add one:

1. Fork the repository
2. Add your direction to `SKILL.md` under "Choose Bold Aesthetic Direction"
3. Provide clear description and when to use it
4. Include code example if possible
5. Submit a pull request

**Good aesthetic direction:**
- Has a distinctive name
- Clear visual characteristics
- Specific font/color recommendations
- Real use cases

### 3. Create Transformation Examples
Show before/after transformations:

1. Add to `references/transformation-examples.md`
2. Include full HTML/CSS code
3. Explain the design decisions
4. Show the memorable feature

### 4. Improve Reference Guides
Help make our references better:

- **Typography** - Add font pairings, new techniques
- **Animation** - Share animation patterns
- **Color** - Contribute color systems
- **Layout** - Show creative layout solutions

### 5. Share Your Designs
Created something amazing with this skill? We'd love to see it!

- Share screenshots in Discussions
- Write about your experience
- Help others learn

## Code Standards

### For Reference Files (Markdown)

```markdown
# Clear Section Headers

Use examples that are:
- Production-ready
- Well-commented
- Accessible
- Responsive

Avoid:
- Generic examples
- Incomplete code
- Unexplained patterns
```

### For Code Examples

- **Semantic HTML5** - Use proper elements
- **CSS Best Practices** - Custom properties, BEM or utility classes
- **Accessibility** - WCAG AA minimum
- **Performance** - Optimize animations, images
- **Responsive** - Mobile-first approach

## Pull Request Process

1. **Fork & Branch**
   ```bash
   git clone https://github.com/YOUR_USERNAME/frontend-design.git
   cd frontend-design
   git checkout -b feature/your-feature-name
   ```

2. **Make Changes**
   - Follow code standards
   - Test your examples
   - Update documentation if needed

3. **Commit**
   ```bash
   git add .
   git commit -m "feat: Add Y2K aesthetic direction with examples"
   ```

   **Commit message format:**
   - `feat:` - New feature
   - `fix:` - Bug fix
   - `docs:` - Documentation only
   - `refactor:` - Code refactoring
   - `test:` - Adding tests

4. **Push & PR**
   ```bash
   git push origin feature/your-feature-name
   ```
   Then create a Pull Request on GitHub

5. **PR Checklist**
   - [ ] Clear description of changes
   - [ ] Code follows standards
   - [ ] Examples are tested
   - [ ] Documentation updated
   - [ ] No breaking changes (or clearly noted)

## Testing Your Contributions

Before submitting:

1. **Visual Review**
   - Does it look good across devices?
   - Is the aesthetic clear and distinctive?
   - Does it have one memorable feature?

2. **Code Quality**
   - Does it work as expected?
   - Is it accessible?
   - Is it performant?

3. **Documentation**
   - Are examples clear?
   - Are decisions explained?
   - Is it easy to understand?
